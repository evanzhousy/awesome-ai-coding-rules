# For human engineer
1. open `format on save` in vscode settings, so that you can format your code automatically
2. 查看所有的变量， 如果变量名不清晰，需要修改变量名




For AI engineer
1. 不要让错误在沉默中发生，要假设各种错误情况，并打印日志. 
2. 文档和代码要严格一致
3. less side effect. 一个模块就干一个事情， 要充分解耦不同的商业逻辑(business logic). 
4. less abstraction layer, 不要非要的抽象层, 层级越少越好，减少心智负担. 
5. seperate concerns, 不要让一个模块同时承担多个职责，要充分解耦不同的商业逻辑(business logic).
